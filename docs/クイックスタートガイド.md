# クイックスタートガイド

このプロジェクトをGitHubに接続する手順です。

## 現在の状態

- ✅ ローカルGitリポジトリは初期化済み
- ✅ プロジェクトファイルは作成済み
- ⏳ リモートリポジトリ（GitHub）への接続が必要

## 接続手順

### 1. リモートリポジトリを接続

```bash
git remote add origin https://github.com/dendrobium0083/img-resizer.git
```

### 2. 接続を確認

```bash
git remote -v
```

以下のように表示されれば成功です：
```
origin  https://github.com/dendrobium0083/img-resizer.git (fetch)
origin  https://github.com/dendrobium0083/img-resizer.git (push)
```

### 3. ブランチ名をmainに変更（必要に応じて）

```bash
git branch -M main
```

### 4. ファイルをステージング（まだの場合）

```bash
git add .
```

### 5. 初回コミット（まだの場合）

```bash
git commit -m "初回コミット: プロジェクトの初期設定"
```

### 6. GitHub認証の設定（重要！）

GitHubはパスワード認証を廃止しているため、**Personal Access Token (PAT)** が必要です。

#### 6-1. Personal Access Tokenを作成

1. GitHubにログイン
2. 右上のプロフィール画像をクリック → **Settings**
3. 左メニューの一番下 → **Developer settings**
4. **Personal access tokens** → **Tokens (classic)**
5. **Generate new token** → **Generate new token (classic)**
6. **Note**: 用途を記入（例：`img-resizer project`）
7. **Expiration**: 有効期限を選択（例：90 days または No expiration）
8. **Select scopes**: `repo` にチェック（リポジトリへのアクセス権限）
9. 一番下の **Generate token** をクリック
10. **表示されたトークンをコピー**（この画面を閉じると二度と表示されません！）

#### 6-2. トークンを使用してプッシュ

```bash
git push -u origin main
```

**ユーザー名を聞かれたら**: GitHubのユーザー名（`dendrobium0083`）を入力  
**パスワードを聞かれたら**: 先ほどコピーした**Personal Access Token**を貼り付け

初回のみ`-u`オプションが必要です。2回目以降は`git push`だけでOKです。

#### 6-3. 認証情報を保存（オプション）

毎回トークンを入力するのが面倒な場合は、Git Credential Managerを使用できます：

```bash
git config --global credential.helper manager-core
```

これにより、次回から自動的に認証情報が使用されます。
